I met the basic requirements to the brief as follows:



  1. Collect and prepare data

    • I used a public dataset and extracted its data into python.
    • I then cleaned this data using pandas to prepare it for my analysis
    • I then stored this clean data in my Firebase Realtime database

  2. Analytics and visualisation

    • I used python to analyse my data and return to me two graphical visualisations of the results using the module matplotlib.
    • These charts show two completely different areas of information in my dataset and are appropriately labelled

  3. Create a basic interactive system interface

    • I developed a web interface that displays my dataset’s information and caters to the interests of the user.
    • This web interface consists of two pages

I met the advanced requirements to the brief as follows:



  1. Ensure that users can interact with the data

    • Users can select from a list of variables that alter the display of the visual graphs they are presented.
    • The user can submit a value and the graph will shorten or lengthen accordingly

  2. Create a form or poll relating to the chosen dataset

    • My form collects personal and relevant information from the user
    • It then presents the user with a list of relevant averages in a variety of different categories

  3. Create a separate area on your information system that makes recommendations

    • I created a section beneath my main information where the user can select from a checklist of tags and receive a list of game recommendations.
    • The website will also alert the user if nothing is selected.

If I had more time, I would have focused on my website’s appearance and created a separate css file for my stylisation. Due to time constraints, my website has a very basic stylisation. I would have decorated my website better to be more visually appealing to the user. I also would’ve added more interactability options like a game quiz that assigns you a game recommendation based on trivial questions about your personality or focused more on some of my original ideas like the ability to receive and store reviews and suggestions, and filter and display different games by developers, compatibility with MAC or Linux, ect. My website idea was too complex and if I had narrowed down my features earlier on in the project, I could’ve devoted an appropriate amount of time to each one and ensured they were the best quality I could provide.
